# Модель метро 82 (CLI, Python)

Учебная программная система для предметной области городского общественного транспорта (метро), реализованная по требованиям из `req.md`.

## Реализованные сущности

- станция метро (`MetroStation`)
- поезд (`Train`)
- билет (`Ticket`)
- турникет (`Turnstile`)
- платформа (`Platform`)
- расписание (`Schedule`, `ScheduleEntry`)
- пассажир (`Passenger`)

## Реализованные операции

- продажа билетов
- посадка и высадка пассажиров
- обслуживание поездов (отправка/возврат)
- операции безопасности (проверка и управление состоянием турникета)
- поддержание расписания (добавление и сдвиг записей)

## Архитектура

- `metro/models.py` - доменные сущности
- `metro/exceptions.py` - доменные исключения
- `metro/repository.py` - in-memory хранилище
- `metro/services.py` - бизнес-операции
- `metro/system.py` - фасад приложения
- `metro/cli.py` - русскоязычный CLI
- `main.py` - точка входа

Решение использует:

- аннотации типов
- обработку исключений для ошибочных ситуаций
- разделение ответственности (чистая архитектура, поддерживаемость)

## Запуск

```bash
python main.py
```

## Запуск unit-тестов

```bash
python -m unittest discover -s tests -v
```

## Особенности обработки ошибок

- невалидный ввод в CLI не завершает программу, а запрашивает ввод повторно
- ошибки порядка действий (например, посадка без билета/турникета) отрабатываются доменными исключениями
- ошибки ссылочной целостности (неверные ID сущностей) также перехватываются и выводятся пользователю понятным сообщением
